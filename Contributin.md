# Normas del repositorio

## Ramas
- Cada miembro del equipo trabajará en su propia rama.
- Nadie puede subir cambios directamente a main.
- Para unir cambios a main hay que abrir un Pull Request y esperar la aprobación de Héctor.

## Nombres de las ramas
- rama-hector
- rama-ismael
- rama-jesus

## Commits
- El mensaje del commit debe describir claramente qué se ha cambiado.
- Ejemplos correctos: "Añadido endpoint de pedidos", "Corregido error en carrito"
- Ejemplos incorrectos: "cambios", "arreglado", "asdf"

## Pull Requests
- Antes de abrir un Pull Request asegúrate de que tu código funciona correctamente.
- Describe brevemente qué has cambiado en el Pull Request.
- Espera la aprobación antes de hacer el merge.

## Actualización
- Antes de empezar a trabajar siempre haz git pull origin main para tener la versión más actualizada.
